## 這個專案的特有經驗

### [2026-04-16] PowerShell 5.1 腳本含中文字元必須存為 UTF-8 with BOM

- scope: project
- project: SCS_SecurityCheck (dotnet-sast-pipeline)
- area: encoding / delivery
- symptom: 打包後同事執行 `scan.ps1` 出現 `TerminatorExpectedAtEndOfString`；here-string 終止符 `"@` 後被判斷為語法錯誤。
- root_cause: Claude Code 的 Write/Edit 工具預設輸出 UTF-8 without BOM。PowerShell 5.1 沒有 BOM 時以系統 ANSI 代碼頁（CP950）解讀腳本，中文多位元組字元被誤讀，產生孤立的單引號，打壞 here-string 語法。
- wrong_assumption: 以為只要把中文移到獨立的 `.txt` 檔案就能解決；但 `scan.ps1` 本身仍有中文 `Write-Host` 字串，一樣會損壞。
- checks_next_time: 任何含中文字元的 `.ps1` 腳本，交付前確認前 3 bytes 是否為 `0xEF 0xBB 0xBF`（UTF-8 BOM）。用 PowerShell 確認：`$b = [System.IO.File]::ReadAllBytes('scan.ps1'); '{0:X2} {1:X2} {2:X2}' -f $b[0],$b[1],$b[2]`
- fix: 用 `[System.IO.File]::WriteAllText(path, content, [System.Text.UTF8Encoding]::new($true))` 重新儲存，加入 BOM。
- rule: 凡含中文字元的 PowerShell 腳本，必須以 UTF-8 with BOM 儲存後再打包交付；打包腳本（`pack_delivery.py`）應在加入 ZIP 前自動驗證 BOM。

### [2026-04-20] SAST 掃描工具不得在原專案目錄注入 PackageReference 或跑 dotnet restore

- scope: project
- project: SCS_SecurityCheck (dotnet-sast-pipeline)
- area: delivery / analyzer-injection / build-isolation
- symptom: 同事執行 v2 `scan.ps1` 後回報「掃描會異動專案的引用元件，導致程式無法重新編譯」；在 Visual Studio 打開專案出現引用混亂、NuGet 還原狀態與 `.csproj` 宣告不一致。
- root_cause: v2 腳本直接在 `$ProjectPath` 根目錄寫入 `Directory.Build.props` 注入 `SecurityCodeScan.VS2019` 與 `Microsoft.CodeAnalysis.NetAnalyzers`，接著對原專案執行 `dotnet restore` + `dotnet build --no-incremental`。雖然掃完會刪除或還原 props 檔，但 `obj/project.assets.json`、`obj/*.csproj.nuget.g.props/.targets`、`packages.lock.json` 與 `bin/` 都已被改寫，**這些副作用不會被還原**。此外 `Directory.Build.props` 向下遞迴生效，會一併污染同目錄底下的所有子專案。
- wrong_assumption: 以為「掃完刪掉 props 檔案」= 原專案完全復原；實際上 NuGet restore 與 build 的輸出副作用遠超單一 props 檔。
- checks_next_time:
  - 任何第三方分析工具若需要注入 PackageReference / props / 編譯，**預設應在專案副本上操作**（`%TEMP%`、`/tmp` 或獨立 workspace），而非原始工作樹。
  - 若因效能或需求必須在原專案操作，必須備份並還原所有輸出：`obj/`、`bin/`、`packages.lock.json`、`Directory.Build.props`、NuGet 相關 `.g.props/.g.targets`。
  - 交付給同事前，先拿一個有 uncommitted changes 的專案本地跑過，確認 `git status` 在掃描前後差異為空。
- fix: v3 改為 `robocopy $ProjectPath %TEMP%\scs-scan-<timestamp>`（排除 `bin/obj/.vs/.git/node_modules/packages/TestResults`），注入、restore、build 全部對副本操作，結束時 `Remove-Item $TempRoot -Recurse -Force`。原專案完全 read-only。
- rule: SAST / Lint / 任何靜態分析交付工具，一律以「原專案 100% read-only」為設計前提；凡需要注入 PackageReference、寫入 props/targets、或跑 `dotnet restore`/`npm install`/`pip install` 的流程，必須在專案副本上進行，並於結束時刪除副本，不得在使用者的實際工作樹留下任何 `obj/bin/lock/快取` 變動。
