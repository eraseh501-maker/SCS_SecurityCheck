# OpenSpec: C# 本機弱點掃描與報告工具

## 1. 目標與範圍

### 1.1 目標
- 提供本機可執行的 Web 介面，讓使用者輸入目錄路徑並掃描 C#/.NET 專案。
- 產出繁體中文 Markdown 弱點報告，內容包含風險摘要、證據、修復建議。
- 在不填 API Key 時，僅使用內建規則引擎完成掃描與建議。
- 在填入 OpenAI 或 Claude API Key 時，額外產生「AI 新增建議」，且不重複基礎規則建議。

### 1.2 非目標
- 不掃描 JS、Python 或其他語言。
- 不依賴 Semgrep、CodeQL、SecurityCodeScan CLI 或其他外部工具。
- 不做動態掃描與執行期攻擊模擬。

## 2. 使用者流程

1. 使用者開啟本機 Web 頁面。
2. 輸入專案資料夾路徑。
3. 選擇是否啟用 AI 新增建議。
4. 若啟用，選擇 AI 供應商並填入 API Key（可留空）。
5. 按下掃描，系統執行規則掃描。
6. 顯示掃描結果摘要與 Markdown 報告預覽。
7. 使用者下載 `.md` 報告。

## 3. 功能需求

### 3.1 後端 API
- `POST /api/scans/run`
  - 請求：`projectPath`、`enableAiSuggestions`、`aiProvider`、`apiKey`
  - 回應：摘要、發現清單、Markdown 內容、AI 新增建議清單

### 3.2 前端頁面
- 路徑輸入欄位（必填）。
- 啟用 AI 建議切換。
- AI 供應商欄位（OpenAI / Claude）。
- API Key 欄位（可選，密碼輸入型態）。
- 掃描按鈕。
- 報告預覽區與下載按鈕。

### 3.3 規則引擎
- 內建至少 10 條 C# 安全規則。
- 規則輸出包含：規則代碼、標題、風險等級、檔案、行號、證據、修復建議。

## 4. 規則清單（MVP）

1. SQL Injection 風險（字串拼接 SQL 或不安全 Raw SQL）。
2. 硬編碼密碼/金鑰。
3. 弱雜湊與弱加密（MD5/SHA1/DES/RC2）。
4. 不安全反序列化（BinaryFormatter 等）。
5. 路徑穿越風險。
6. 開放重新導向。
7. SSRF 風險（外部輸入直接發送 HTTP）。
8. 缺少授權保護（敏感端點無授權屬性）。
9. 過度寬鬆 CORS 設定。
10. 例外細節外洩（回傳堆疊/內部錯誤）。

## 5. AI 新增建議需求

### 5.1 觸發條件
- 只有在 `enableAiSuggestions=true` 且 `apiKey` 有值時才呼叫 AI。

### 5.2 輸出要求
- AI 回覆需轉為結構化建議清單。
- 每項建議需標示為 `AI 新增建議`。
- 與基礎規則建議做去重（文字正規化後比對）。
- 若 AI 建議與基礎規則重複，必須排除。

## 6. 非功能需求

- 本機執行，不需安裝外部掃描工具。
- 預設僅掃描 `.cs` 與 `.csproj`。
- API Key 不寫入日誌、不落地保存。
- 錯誤訊息不得洩漏敏感系統資訊。

## 7. 報告格式（繁體中文 Markdown）

1. 掃描摘要
2. 風險統計
3. 重大弱點清單
4. 修復建議
5. AI 新增建議（僅可用時）
6. 附錄（略過檔案、限制說明）

## 8. 驗收標準

- 可掃描指定本機 C# 專案並產出繁中 Markdown。
- 無 API Key 時，仍可完整產出基礎報告。
- 有 API Key 時，報告中出現 AI 新增建議章節。
- AI 新增建議不重複基礎規則建議。
- 同事在本機執行不需 skills 封包與外部掃描器。
